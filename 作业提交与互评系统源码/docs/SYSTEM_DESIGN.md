# 作业提交与互评系统设计说明

## 1. 总体结构

系统采用 Java Swing 桌面应用实现，入口为 `App`，启动后进入 `RoleSelectionFrame` 角色选择界面。

- 教师端：`TeacherFrame`，负责作业管理、数据监控、成绩管理。
- 学生端：`StudentFrame`，负责作业查看、作业提交、评测与互评、成绩查询。
- 数据层：`DataStore`，负责作业、提交、互评记录的读写、统计和事件通知。
- 服务层：`AutoJudgeService`、`PlagiarismService`、`TextExtractor`，分别负责自动评测、查重和文本抽取。
- 权限上下文：`RoleSession`，区分 `TEACHER` 与 `STUDENT`，界面操作必须携带对应会话调用数据层。

## 2. 教师端模块

### 作业管理

- 发布作业：填写标题、描述、截止日期。
- 配置评测：编程作业可配置样例输入和期望输出。
- 编辑作业：修改已有作业信息。
- 删除作业：删除作业时级联删除对应提交记录和互评记录。

### 数据监控

- 提交概览：统计每个作业的提交人数、未提交人数和总人数。
- 查重管理：对指定作业的所有提交进行相似度比较，并标记高风险提交。
- 评测概览：按作业统计通过、失败、编译错误和其他状态数量。

### 成绩管理

- 互评管理：查看学生互评分数、评语、争议状态和仲裁状态。
- 争议仲裁：教师可调整争议互评的分数和评语，保存后清除争议并标记已仲裁。
- 最终评分：系统按 `自动评测 60% + 互评均分 40%` 计算最终分。

## 3. 学生端模块

### 作业查看

- 查看所有未截止作业。
- 查看作业标题、描述、截止日期、类型、互评状态、样例输入和期望输出。

### 作业提交

- 学生只能以当前登录身份提交自己的作业。
- 支持上传 `.java` 源文件和 `.docx` 文档。
- 编程作业强制使用 `.java` 文件，并在提交后触发自动评测。
- 已截止作业不能提交。

### 评测与互评

- 学生可查看自己提交记录的自动评测状态、信息和自动分。
- 互评阶段由系统随机分配其他学生的匿名提交。
- 学生不能互评自己的提交，也不能重复评价同一提交。

### 成绩查询

- 查看自己的最终得分、自动评测结果、互评均分和同学评语。

## 4. 数据一致性

`DataStore` 维护 `DataChangeListener` 列表。作业、提交、互评、查重、仲裁等数据变化后会调用 `notifyChanged()`，教师端和学生端界面会自动刷新表格和下拉框，保证两端数据同步显示。

## 5. 权限控制

系统通过 `RoleSession` 明确记录当前会话角色。

- 教师会话才能发布、编辑、删除作业，触发查重和仲裁互评。
- 学生会话才能提交作业、领取互评任务和提交互评。
- 学生提交和互评时会校验当前学生身份，不能伪造其他学生学号。
- 教师端界面不提供学生提交入口，学生端界面不提供教师管理入口。

## 6. 持久化与运行

数据以 TSV 文件保存在运行目录下的 `data` 文件夹中，提交文件保存在 `data/submitted`。

编译：

```bash
javac -encoding UTF-8 *.java
```

从 `src` 目录运行：

```bash
java com.yuzhen.assignment.App
```
